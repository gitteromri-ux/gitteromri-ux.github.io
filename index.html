Method & rank logic

How the rank is assigned

  1. For each Sunday, look back 10 weeks per Employee × School × Language.
  2. Compute RegFromCalls = Reg / Calls — this is the single rank source used everywhere on this dashboard.
  3. Within each (Date, School, Language) group, sort by ratio and assign a volume-weighted quartile: ≤25% → R1, ≤50% → R2, ≤75% → R3, >75% → R4.
  4. Rank 1 = weakest, Rank 4 = strongest.
  5. Both % calls to R3+R4 and % contacts by R3+R4 bucket reps by this same RankRegFromCalls — the only difference is whether we count calls or contacts in the numerator.

How the goal is computed

  1. For each school × language combo, find every month's actual SOV.
  2. Take the top 25% of months by SOV — the months that actually made revenue.
  3. The goal % calls to R3+R4 = average % R3+R4 calls in those top months.
  4. Same for contacts: goal % contacted to R3+R4 = avg % R3+R4 contacts in top months.
  5. Combos with < 500 calls in the period are flagged low confidence.

What this avoids

  • Doesn't assume "more R4 = always better" — the goal comes from your own good months.
  • Doesn't penalize R1/R2/R3 for being R1/R2/R3 — only the allocation % matters.
  • Won't declare a thin combo "broken" — sample-size guardrail at 500 calls.
  • Doesn't hard-code one number across all schools — every combo gets its own goal.

How to read a row
